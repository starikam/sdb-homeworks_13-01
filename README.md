  ### Задание 2

Проводим сканирование с помощью команд:

  nmap -sS 192.168.1.10   # SYN scan
  nmap -sF 192.168.1.10   # FIN scan
  nmap -sX 192.168.1.10   # Xmas scan
  nmap -sU 192.168.1.10   # UDP scan

Чем отличаются режимы с точки зрения трафика:

1)	SYN-сканирование (-sS)
   
Самый распространённый режим. Отправляет только первый пакет TCP-рукопожатия, соединение никогда не завершается полностью.
Быстрый режим, не оставляет записей в логах приложений (соединение не устанавливается).

2)	FIN-сканирование (-sF)
Отправляет пакет с флагом FIN, как конец соединения но без предварительного установления связи. 
По  стандарту закрытый порт обязан ответить RST на неожиданный FIN, а открытый проигнорирует.
  
3)	Xmas-сканирование (-sX)
Отправляет пакет с тремя флагами одновременно: FIN + PSH + URG 
Логика ответов идентична FIN-скану. Разница только в наборе флагов в исходящем пакете.

4)	UDP-сканирование (-sU)
UDP – принципиально другой протокол, он не имеет механизма установки соединения. Очень медленный, с помощью этого сканирования сложно отличить открытый порт от фильтрованного.

Табличка, как отвечает сервер:
| Режим | Флаги пакета | Открытый порт | Закрытый порт |
  |-------|-------------|---------------|---------------|
  | SYN | SYN | SYN+ACK | RST+ACK |
  | FIN | FIN | Молчание | RST+ACK |
  | Xmas | FIN+PSH+URG | Молчание | RST+ACK |
  | UDP | - | Молчание / UDP-ответ | ICMP unreachable |
